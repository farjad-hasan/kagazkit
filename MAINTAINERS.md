# Maintainers

KagazKit is currently maintained by a single primary maintainer.

## Current Maintainer

- Farjad Hasan ([farjad-hasan](https://github.com/farjad-hasan))

## Maintainer Model

- This is an honest single-maintainer project today. There is no backup maintainer or release co-owner yet.
- Pull requests, issues, and releases are handled on a best-effort basis.
- Operational knowledge should live in repository documentation rather than private maintainer context.

## Maintainer Responsibilities

- review and merge pull requests
- triage issues and bug reports
- cut GitHub releases and publish to PyPI
- respond to security reports according to [SECURITY.md](SECURITY.md)
- keep CI, packaging, and release instructions current

## Response Expectations

- security reports: acknowledgement within 48 hours
- standard issues and pull requests: best effort, typically within 7 days

## Path to Shared Stewardship

KagazKit is open to additional maintainers over time. Until then:

- contributor-facing processes should stay lightweight and documented
- release steps should remain reproducible from [RELEASE.md](RELEASE.md)
- support and reporting paths should remain clear in [SUPPORT.md](SUPPORT.md) and [README.md](README.md)
